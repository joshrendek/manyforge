# US6 — MCP Host Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax. This plan is written for a FRESH session with zero prior context — but you have the full repo. Where a task says "mirror `<file>`", open that file and follow its pattern exactly; those are the canonical conventions.

**Goal:** Let an agent use tools from external MCP servers (remote Streamable-HTTP), discovered at run start, registered namespaced + fail-closed (External), and routed through the existing RBAC → gate → approval → outbox → executor → audit path.

**Architecture:** A new domain-agnostic `internal/platform/mcp` package (JSON-RPC 2.0 client over an SSRF-guarded HTTP transport, + a mock) mirrors `internal/platform/ai`. A per-business `mcp_server` registry (sealed auth, mirrors `ai_provider_credential`) + per-agent opt-in (`agent.allowed_mcp_servers`). At `Engine.Execute` start, opted-in servers are connected + `tools/list`'d; each tool registers as `mcp:<server>:<tool>`, `Effect=External`, `RequiredPerm="mcp.invoke"`. Execution reuses the US4 `ApprovalExecutor`, extended with an `mcp:` routing branch.

**Tech stack:** Go, pgx/v5, sqlc (reads `db/schema.sql`), golang-migrate (`migrations/NNNN_*.{up,down}.sql`), testcontainers, chi, the in-house `netsafe`/`crypto.Sealer`/`events`/`httpx` packages.

**Authoritative design:** `docs/superpowers/specs/2026-06-04-us6-mcp-host-design.md`. Read it first. **bd:** issue `manyforge-1ah` (US6) under epic `manyforge-deo`; claim it before starting.

**Migration numbering:** the latest on disk is **0035** (`ls migrations/` to confirm). US6 uses **0036** (mcp_server + agent column) and **0037** (mcp.invoke permission).

**Pre-flight every session:** `export PATH="$PATH:$HOME/go/bin"` (golangci-lint lives there; without it `make lint` silently runs vet-only). Docker/Colima must be up for `int-test`. After editing `db/query/*.sql`, run `make generate` (never hand-edit `internal/platform/db/dbgen`). gopls reports stale `undefined: dbgen.X` on new dbgen references — trust `go build`, not the IDE.

```go
// blockedWith reports whether ip must be refused. allowLoopback permits 127/8 + ::1
// ONLY (for dev MCP servers); all other private/link-local/metadata stay blocked.
func blockedWith(ip net.IP, allowLoopback bool) bool {
	if ip == nil {
		return true
	}
	if ip.IsLoopback() {
		return !allowLoopback
	}
	if ip.IsPrivate() || ip.IsLinkLocalUnicast() || ip.IsLinkLocalMulticast() ||
		ip.IsUnspecified() || ip.IsMulticast() {
		return true
	}
	for _, m := range metadataIPs {
		if ip.Equal(m) {
			return true
		}
	}
	return false
}

// Blocked reports whether ip is a destination outbound requests must refuse
// (loopback blocked — the default, locked-secure posture).
func Blocked(ip net.IP) bool { return blockedWith(ip, false) }

// Options configures a guarded client.
type Options struct{ AllowLoopback bool }

// NewClientWithOptions builds a guarded client; AllowLoopback permits loopback only.
func NewClientWithOptions(timeout time.Duration, o Options) *http.Client {
	dialer := &net.Dialer{Timeout: 10 * time.Second}
	return &http.Client{Timeout: timeout, Transport: &http.Transport{
		DialContext: func(ctx context.Context, network, addr string) (net.Conn, error) {
			host, port, err := net.SplitHostPort(addr)
			if err != nil {
				return nil, err
			}
			ips, err := net.DefaultResolver.LookupIPAddr(ctx, host)
			if err != nil {
				return nil, err
			}
			for _, ip := range ips {
				if blockedWith(ip.IP, o.AllowLoopback) {
					return nil, fmt.Errorf("blocked address %s for host %s", ip.IP, host)
				}
			}
			return dialer.DialContext(ctx, network, net.JoinHostPort(ips[0].IP.String(), port))
		},
	}}
}
```
Refactor `NewClient` to delegate: `func NewClient(timeout time.Duration) *http.Client { return NewClientWithOptions(timeout, Options{}) }`.

## Task 3: the `mcp` package — schema + JSON-RPC client + mock

**Files:** Create `internal/platform/mcp/{schema.go,client.go,mock.go,client_test.go,mock_test.go}`

Mirror `internal/platform/ai/{schema.go,anthropic.go,mock.go}` for structure (injected `*http.Client`, `url.Parse(...).JoinPath`, `http.NewRequestWithContext`, `io.LimitReader` body cap, never surface upstream bodies).

**MCP wire protocol (JSON-RPC 2.0 over Streamable-HTTP, spec 2025-06-18):** all requests are `POST <server url>` with `Content-Type: application/json` and `Accept: application/json, text/event-stream`. An `Mcp-Session-Id` response header from `initialize` MUST be echoed on subsequent requests. Responses are either `application/json` (one JSON-RPC object) or `text/event-stream` (SSE — scan `data:` lines for the JSON-RPC message whose `id` matches).
- `initialize` → params `{"protocolVersion":"2025-06-18","capabilities":{},"clientInfo":{"name":"manyforge","version":"0"}}`; then POST notification `{"jsonrpc":"2.0","method":"notifications/initialized"}`.
- `tools/list` → `result.tools[]` each `{name, description, inputSchema}`.
- `tools/call` → params `{"name":..., "arguments":<obj>, "_meta":{"idempotencyKey":"<hint>"}}`; result `{"content":[{"type":"text","text":...}], "isError":bool}`.

- [ ] **Step 1: `schema.go`** — types (no test needed; types only):
