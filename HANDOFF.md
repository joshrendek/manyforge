## Gotchas (don't relearn these)
- **gopls LIES persistently** on every file referencing `dbgen` ("undefined: dbgen.X") and on every `//go:build integration` file ("No packages found"). These are FALSE POSITIVES ‚Äî `go build ./...` is clean and tests pass. **Trust `go build`/`go test`, NEVER the IDE diagnostics panel.**
- **dbgen nullable nuance:** with the `uuid`/`timestamptz` go_type overrides, `emit_pointers_for_null_types` does NOT apply to them. `<col> uuid NULL` ‚Üí `pgtype.UUID` (NOT `*uuid.UUID`); `text NULL` ‚Üí `*string`; `timestamptz NULL` ‚Üí `pgtype.Timestamptz`. Plain `text NOT NULL` ‚Üí `string`. (`ticket.ConnectorID` is `pgtype.UUID`, `ticket.ExternalID`/`ExternalUrl` are `*string`.)
- **Two casings:** Go service structs use `BaseURL`/`AllowPrivateBaseURL`; generated dbgen fields use `BaseUrl`/`AllowPrivateBaseUrl` (Url, not URL). Match each layer.
- **Composite-FK discipline:** every new child table needs `business_id`+`tenant_root_id`, `FOREIGN KEY (x_id, tenant_root_id) ‚Üí parent(id, tenant_root_id)`, a `*_troot_immutable` trigger, RLS (`authorized_businesses(current_principal())`), GRANT to `manyforge_app`.
